Product Requirements Document (PRD)
Product Name: AlgoTrade Pro Engine - Futures + Spot Edition (V1)
Target User: Algorithmic traders managing multiple Binance Futures and Binance Spot accounts using TradingView strategies.

1. Product Overview
AlgoTrade Pro is a local, self-hosted SaaS application that bridges TradingView chart algorithms to Binance accounts. The original Binance Futures workflow remains the core behavior. Spot Trading is an additional feature that can run alongside Futures without replacing it.

The product must make Futures and Spot visually distinct across the UI. Users must always know whether an account, mapping, trade, position, holding, webhook result, or setting belongs to Futures or Spot.

2. Core Components

2.1. The Web Dashboard (UI)
The control center for the trader.
* Market Identity: The sidebar and page headings show "Futures + Spot". Individual rows/cards show either "Futures" or "Spot" badges.
* Real-Time Data: Displays aggregated account equity/wallet balance, available capital, and utilization/deployed percentage across all active Futures and Spot accounts.
  - Futures accounts use walletBalance, availableMargin, and margin utilization from Binance Futures.
  - Spot accounts use estimated Spot equity, available USDT, and deployed capital.
* Account Management: Users can add, edit, and delete Binance API credentials. Each account has market_type = "futures" or "spot".
  - Existing accounts default to Futures for backward compatibility.
  - Futures accounts keep leverage, LONG/SHORT/EXIT behavior, stoploss, trailing stop, and position management.
  - Spot accounts are 1x only and use BUY/SELL inventory behavior.
* Symbol Mappings: Mappings inherit the account market type and validate symbols against the correct Binance market.
* Positions & Holdings: One page shows Futures open positions and Spot holdings together.
  - Futures rows show LONG/SHORT, size, entry price, mark price, notional, leverage, unrealized P&L, and P&L %.
  - Spot rows show HOLD, held/free/locked quantity, last price, estimated value, Spot 1x, and P&L placeholder.
* Order History: Tracks both Futures and Spot trades with a Market column.
  - Futures actions: LONG, SHORT, EXIT.
  - Spot actions: BUY, SELL, EXIT.
  - Futures rows show leverage. Spot rows show Spot 1x.
* Webhook Logs: Shows market result as Futures, Spot, or Mixed when one signal routes to both market types.
* Testnet Mode: One toggle controls Binance Futures Testnet and Binance Spot Testnet. Banner reads "TESTNET MODE - Futures and Spot trading with fake money".

2.2. TradingView Strategy
The TradingView strategy remains the signal source.
* Webhook JSON format: {"symbol":"BTCUSDT","action":"LONG|SHORT|EXIT|BUY|SELL","timeframe":"5m","price":65000.0}
* Futures mapping:
  - LONG, ENTRY, or BUY opens a Futures long.
  - SHORT opens a Futures short.
  - EXIT or SELL closes any open Futures position.
* Spot mapping:
  - LONG, ENTRY, or BUY places a Spot market BUY.
  - SHORT, EXIT, or SELL sells the owned Spot base asset.
* The same symbol/timeframe can route to both Futures and Spot if the user creates mappings on accounts of both market types.

2.3. API Server
* Signal Ingestion: /api/webhook accepts TradingView JSON payloads.
* Symbol Normalization: Strips exchange prefixes, .P suffixes, and PERP suffixes before routing.
* Timeframe Normalization: Converts user-friendly mappings like 5m, 1h, 4h, and 1D to TradingView formats.
* Market-Aware Routing: The API resolves account mappings by symbol, timeframe, and account market type.
* Market-Aware Validation: Futures mappings validate against Binance Futures exchangeInfo; Spot mappings validate against Binance Spot exchangeInfo.

2.4. Bot Engine
The Bot Engine is market-aware.
* Shared Risk Evaluation: bot_active, daily_loss_limit, and max_drawdown apply before both Futures and Spot trades.
* Futures Execution:
  - Uses Binance Futures account, exchange info, ticker, leverage, position, and order endpoints.
  - Supports LONG, SHORT, EXIT, single/multi trade mode, leverage, stoploss, and trailing stop.
  - Single mode closes an existing Futures position on the symbol before a new entry.
  - Uses reduceOnly=True when force-closing or exiting positions.
* Spot Execution:
  - Uses Binance Spot account, exchange info, ticker, and order endpoints.
  - BUY/ENTRY/LONG places a Spot market BUY using fixed USDT or percent of estimated Spot equity, capped by available USDT.
  - SELL/EXIT/SHORT sells the available base asset.
  - Spot is always 1x. Futures leverage settings do not apply.
  - Single mode blocks duplicate buys while already holding the base asset. Multi mode allows scale-in buys.
  - Stoploss attempts a Spot STOP_LOSS_LIMIT sell. Trailing stop is Futures-only and ignored for Spot with a warning.
* Ledger Recording: Trade records include market_type so history can distinguish Futures from Spot.

2.5. Per-Account Trading Settings
Each account has:
* market_type: "futures" or "spot".
* trading_size_type: "percent" or "fixed".
* trading_size_value: percentage or fixed USDT amount.
* leverage: Futures leverage 1x-125x. Forced to 1 for Spot.
* stoploss_percent: Futures STOP_MARKET or Spot STOP_LOSS_LIMIT behavior.
* trail_activation_pct and trail_callback_pct: Futures trailing-stop controls only.
* trade_mode: Futures single/multi position behavior; Spot single/multi scale-in behavior.

2.6. Bot Settings Defaults
* Global defaults continue to seed new accounts.
* default Futures leverage remains configurable.
* Spot accounts ignore leverage defaults and use 1x.
* Existing accounts are not retroactively changed when defaults are updated.

3. Functional User Flows

3.1. Adding an Account
1. User chooses Market Type: Futures or Spot.
2. User enters Name, API Key, and API Secret.
3. Backend verifies credentials against the selected market.
4. Account is saved encrypted, with a market-specific enabled badge.
5. Account settings are created from global defaults; Spot leverage is forced to 1.

3.2. Adding a Symbol Mapping
1. User adds a mapping under a specific account.
2. Backend validates the symbol against that account's market.
3. Mapping stores the inherited market_type.

3.3. Automated Futures Trade
1. TradingView sends LONG/SHORT/EXIT for a mapped Futures account.
2. Bot checks risk limits.
3. Bot applies Futures leverage and single/multi mode.
4. Bot executes Futures market order and optional Futures stop/trailing orders.
5. UI updates Futures positions, balances, history, and logs.

3.4. Automated Spot Trade
1. TradingView sends BUY/LONG/ENTRY or SELL/SHORT/EXIT for a mapped Spot account.
2. Bot checks risk limits.
3. BUY calculates USDT allocation and places Spot market BUY.
4. SELL sells available base asset.
5. UI updates Spot holdings, balances, history, and logs.

3.5. Positions & Holdings
1. User opens Positions & Holdings.
2. Futures rows can be force-closed with reduce-only Futures market orders.
3. Spot rows can be sold with Spot market SELL orders.
4. UI labels and confirmation modals clearly distinguish "Close Futures Position" from "Sell Spot Holding".

4. Non-Functional Requirements
* Latency: Webhook ingestion to exchange submission should remain under 500ms where network conditions allow.
* Resilience: Binance API failures must be logged and returned without crashing the API server.
* Security: API secrets encrypted at rest, dashboard protected by sessions, optional webhook secret supported, no hardcoded credentials.
* Backward Compatibility: Existing accounts and mappings default to Futures after migration.
* Testnet Support: Futures and Spot testnets are both supported through the same Testnet Mode toggle.
